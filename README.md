# CNN для классификации изображений на CIFAR-10

Проект для практической работы по компьютерному зрению на TensorFlow/Keras.

В проекте реализованы:
- собственная сверточная нейронная сеть для многоклассовой классификации RGB-изображений;
- обучение и сравнение двух архитектур:
  - `baseline_model` — базовая и более точная;
  - `fast_model` — облегченная и более быстрая;
- расчет метрик качества и FPS;
- сохранение обученной модели;
- демонстрационный веб-интерфейс на **Gradio**;
- упаковка проекта в **Docker**.

## Датасет

Используется **CIFAR-10**:
- 10 классов;
- цветные изображения RGB;
- размер 32x32.

Классы:
- airplane
- automobile
- bird
- cat
- deer
- dog
- frog
- horse
- ship
- truck

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Обучение модели

```bash
python -m src.train
```

После обучения будут сохранены:
- модель: `models/baseline_model.keras`
- таблица сравнения: `results/model_comparison.csv`
- история обучения: `results/*.json`
- графики: `results/*.png`

### 3. Запуск Gradio-интерфейса

```bash
python app.py
```

По умолчанию интерфейс откроется на:

```text
http://127.0.0.1:7860
```

## Запуск через Docker

### Сборка образа

```bash
docker build -t cnn-cifar10-gradio .
```

### Запуск контейнера

```bash
docker run -p 7860:7860 cnn-cifar10-gradio
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:7860
```

## Как использовать проект для отчета

### Что здесь закрывает требования задания

1. **Выбор датасета**  
   Выбран CIFAR-10: более 5 классов, изображения цветные.

2. **Реализация собственной CNN**  
   Используются стандартные библиотечные слои TensorFlow/Keras: `Conv2D`, `MaxPooling2D`, `Dense`, `Dropout`, `BatchNormalization`, `GlobalAveragePooling2D`.

3. **Графики loss, метрики и FPS**  
   В проекте автоматически строятся графики `loss` и `accuracy`, рассчитываются `accuracy`, `F1-macro`, `F1-weighted`, а также скорость инференса в FPS.

4. **Эксперимент с архитектурой**  
   Сравниваются две модели: базовая и ускоренная.

5. **Сравнение моделей**  
   Формируется таблица с параметрами, временем обучения, метриками и FPS.
